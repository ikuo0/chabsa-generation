      - 実装関数名: test_distance_weight()
      - エントリー関数: main()
      - 実行方法: python src/uca002/ucb001/main.py distance_weight
      - 入力 1~100 の連続値

        - 作成時に既にファイルがあれば削除する

      - 出力
        - 前述の distance_weight.tsv を読み込み、matplotlib でグラフ化する

        - 各計算結果、総合のグラフを出力する
        - 出力画像

  - UCB002 chABSAデータをベクトル化する
    - 実装ファイル: $root/src/uca002/ucb002/main.py
    - 実装関数名: vectorize_text()
      - 窓解析パラメータを引数とする
        - 単語幅(w): デフォルト 15 単語
        - スライド幅(s): デフォルト 1 単語スライド
    - エントリー関数: main()
    - 実行方法: python src/uca002/ucb002/main.py
    - 入力 SQLite: $root/data/uca001/ucb003/chabsa_normalized.db (UCB003 を import して利用する)
    - 入力 SentencePieceモデル: $root/data/uca001/ucb006/spm_model.model
      - "/data/uca001/ucb006/spm_model.model" 文字列は関数で取得できるようにする、他処理でも利用する
    - ベクトル化
      - ※窓解析パラメータは引数依存、以下はデフォルト値での説明
      - 文章を単語に分割する
      - 15単語幅、1単語スライドの窓解析を実施
      - 前処理、加工について
        - 先頭にbosを15単語パディングする
        - 末尾にeosを15単語パディングする
        - 文章は bos * 15, eos * 15 と30単語追加された状態となる
      - 方法
      　- 以下、パディング済みを前提とする、先頭、末尾、位置に関する記述は全てパディング後として考えること
        - 16単語取り出し窓解析ループする(窓幅+1)
          - ループインデックス: for(0 to (文章の単語数 - 窓幅 - 1)) { 0～(単語数 - 窓幅 - 1)までの範囲でスライドする }
          - データ1: 文書インデックス
            - SQLiteのrow_idを文書インデックスとする
          - データ2: Yデータ
            - 16単語目の単語インデックスとする
          - データ3: Xデータ
            - 先頭イメージ: [bos, bos, .... 単語]
            - 末尾イメージ: [単語, eos, eos, ....]
            - 文脈ベクトルは距離重み付きONE-HOTの総和をベクトル化の方法とする
            - 列数＝総単語数の0埋めの粗ベクトルを準備する
            - 距離重みは UCB001 で試験した結果、1/(d^0.25) とする
            - 正解データの前15単語について、15単語目は 1/((15-15 + 1)^0.25)、14単語目のインデックスは 1/((15-14 + 1)^0.25) として重み付けした値を単語の該当インデックスに加算していく
      - ＡＩ向け指示
        - ベクトル作成はＡＩでは想定通りに作られないため、空実装とする。
        - 文書ID配列, Yデータ配列, Xデータ配列を空で準備
        - SQLite から１文章のデータを全て読み出し変数に格納するループを作成
          - ベクトル化関数
            - 文書ID配列(npy 1次元配列), Yデータ配列(npy 1次元配列), Xデータ配列(scipy.sparse.csr_matrix 2次元配列) を返却する空関数を作成
          - 戻り値を 文書ID配列, Yデータ配列, Xデータ配列 に追加
        - ループ終了後、文書ID配列, Yデータ配列, Xデータ配列 をそれぞれ返却
        - 呼び出し元で、返却された文書ID配列, Yデータ配列, Xデータ配列 をそれぞれ npy, npz 形式で保存する
    - 出力
      - 出力: 文書インデックス npy 配列
        - $root/data/uca002/ucb002/chabsa_index.npy
        - 作成時に既にファイルがあれば削除する
        - "$root/data/uca002/ucb002/chabsa_index.npy" 文字列は関数で取得できるようにする、他処理でも利用する
      - 出力: Xデータ(文脈ベクトル) scipy.sparse.csr_matrix として保存
        - 粗ベクトル(scipy.sparse.csr_matrix)とする
          - 6000次元～等、大きな次元数となるため、メモリ効率を考慮して疎行列とする
        - $root/data/uca002/ucb002/chabsa_vector.npz
        - 作成時に既にファイルがあれば削除する
        - "$root/data/uca002/ucb002/chabsa_vector.npz" 文字列は関数で取得できるようにする、他処理でも利用する
      - 出力: Yデータ(文脈の次の単語) npy 配列

        - 作成時に既にファイルがあれば削除する
